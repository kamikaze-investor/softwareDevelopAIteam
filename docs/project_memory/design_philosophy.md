# Design Philosophy

**Importance Level: 4**

---

全AIは設計・実装判断の際に必ずこの思想に照らし合わせること。

## 1. スマホ完結
すべての操作がスマホから完結すること。PCを必要とする操作を作らない。

## 2. 全自動優先
人間の操作を最小化する。AIが自律的に判断・実行できる場合は自動化する。

## 3. 承認最小
承認を求めるのは絶対に必要な時だけ。詳細は `docs/project_memory/rules/approval_rules.md` 参照。

## 4. Rollback重視
失敗しても即座に戻せる設計を優先する。小さいコミット・ブランチ運用を徹底する。

## 5. Context重視
AIには必要な情報だけを渡す。情報過多は判断品質を下げる。Context Pack経由で情報を届ける。

## 6. 小さく変更
1タスク = 最小変更単位。大きい変更は必ずサブタスクに分割する。

## 7. 小さくコミット
1タスク = 1コミット。1サブタスク = 1コミット。
